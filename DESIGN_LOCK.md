# Design lock

Исходный zip-архив с Figma-вёрсткой является источником правды для публичного дизайна.

## Правило

Страницы и компоненты, которые были в исходном архиве, нельзя визуально переделывать.

Разрешено:
- подключать реальные данные;
- добавлять серверную логику;
- добавлять авторизацию;
- добавлять проверки доступа;
- исправлять баги без изменения визуального вида;
- аккуратно подключать состояния active/inactive, если они уже предусмотрены дизайном.

Запрещено:
- менять layout;
- менять размеры блоков;
- менять цвета;
- заменять Figma-дизайн на новый;
- переписывать существующие публичные страницы в другом стиле.

Новый дизайн допустим только для новых страниц, которых не было в исходном архиве: например, админка, новые технические страницы, служебные страницы.

## Страницы из исходного архива

- /
- /cooperation
- /library
- /library/base
- /videolecture
- /profile/settings
- /profile/subscription
- /profile/discount
- /login
- общий layout
- header
- footer
- профильный aside
- публичные компоненты главной страницы
- публичные карточки/секции из архива

Перед изменением этих страниц нужно сохранять визуальное соответствие архиву.
